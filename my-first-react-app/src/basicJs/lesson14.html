We can export one or more objects from a given module. These are called named exports.You may say class is not an object. Well in JavaScript, technically a class is an object. Because JavaScript classes are syntactic sugar over constructor functions. And functions as I told you before are objects so a class is technically an object in JavaScript.

There is something called default export in Js. If we have only one object in module, we use default keyword.

Default exports:

import { Person1 } from "./lesson11";
export class default Developer1 extends Person1 {
    constructor(name,age,role) {
        this.name = name;
        this.age = age;
        this.role = role;
    }
    printRole(){
        console.log(this.role);
    }

To import we use ,
import Developer1 from "./lesson12";
Named exports:

import { Person1 } from "./lesson11";

export function hello(){}

export class default Developer1 extends Person1 {
    constructor(name,age,role) {
        this.name = name;
        this.age = age;
        this.role = role;
    }
    printRole(){
        console.log(this.role);
    }

To import we use ,
import {Developer1, hello} from "./lesson12";


Default and named exports:

import { Person1 } from "./lesson11";
export function hello(){}
export class default Developer1 extends Person1 {
    constructor(name,age,role) {
        this.name = name;
        this.age = age;
        this.role = role;
    }
    printRole(){
        console.log(this.role);
    }
To import we use ,
import Developer1, {hello} from "./lesson12";
It is also possible that a module can have both named and default exports in it. React is a good example of it. Let us take a little look about this.

Let us make Developer1 class default export and hello function named export. Now this module contains both named and default exports. So to import named and default exports from this module, our import statement looks like this.

import Developer1, {hello} from "./lesson12";

Now why is this important. You will see lot of this pattern when you are developing react applications. Let us look at an example
import React, {component} from "react";

React is the default export from "react" library and component is the named export . Note, we didn't use "./react". This is because we use './' to modules that we have developed. react is a third party module which is node modules.
