In Javascript we always access properties and methods using dot notation. So, if you want to store a property in a variable we use the following method.

const marks = {
    physics: 75,
    chemistry: 80,
    maths: 82
};
const physics = marks.physics;
const chemistry = marks.chemistry;
const maths = marks.maths;
console.log(physics,chemistry,maths);
We need to extract the value of these properties, and store them in separate variables. So this is what we typically do. We have to access each and every object using dot notation and store it in variables.

So you have to repetitively use marks. to access the property. In modern Javascript we have something called Destructuring that solves this problem.


const marks = {
    physics: 75,
    chemistry: 80,
    maths: 82
};
const {physics:phy, chemistry:chem, maths: mat} = marks; <==
console.log(phy,chem,mat);
We can rewrite the above code like this. Let us see what code marked with <== does. It takes physics, chemistry, maths properties from marks and assign it to variable py, chem, mat. So this is exactly same as
const phy = marks.physics;
const chem = marks.chemistry;
const mat = marks.maths;

It is not necessary that you have to access all property if yoou use this method. If you want one property then you can access that alone. And if name of the variable is exactly same as property that you are accessing then the code becomes more cleaner.
const {physics} = marks;
