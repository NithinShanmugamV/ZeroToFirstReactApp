This keyword works differently in JavaScript. Let us look at it.
const opertions = {
    num1: 10,
    num2:8,
    demoPrint() {
        console.log(this) <==
    }
};
opertions.demoPrint();//1.
const demoPrint= opertions.demoPrint;//2.
console.log(demoPrint);
demoPrint();

When we write operations.demoPrint() (1.), this keyword returns refernce to operations object. But this keyword doesn't work in the same way.

Instead of calling a method let us now refernce a method and assign it to a variable(2.). Note, opertions.demoPrint is not calling method demoPrint. We are just getting refernce to that method. To call we add parantheses opertions.demoPrint() . Still confusing, let us console.log demoPrint constant. Look you get a refernce to that function. Now, what happens when we call this demoPrint constant. We get something different, Why?.

That's why I told you that the this keyword in JavaScript behaves differently from other programming languages. The value of this is determined by how a function is called. If we call a function as a method in an object(opertions.demoPrint();), this will always return a reference to that object.

However, if we call a function as a stand alone object, or outside of an object, this will return the global object which is the window object in browsers. Note here const demoPrint= opertions.demoPrint; we reference a method in object. But when you call it is not considered as calling a method it is considered as calling a funtion, since we referenced a method it is no more considered as method in an object.
